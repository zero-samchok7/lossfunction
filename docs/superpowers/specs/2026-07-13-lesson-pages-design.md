## 1. 전체 구조

```
index.html          ← 5차시 허브 메인 (기존 허브 교체)
lesson1.html        ← 1차시: 추세선 직접 그리기
```

- iPad landscape 최적화 (100dvh, overflow:hidden, 3패널 flex)

**공통 디자인 시스템 (`lesson-common.css`)**
- 모든 lesson1~5.html이 공유하는 단일 CSS 파일
- 색상 팔레트 (CSS 변수):

- 헤더: 높이 48px, 흰 배경, 하단 border, 중앙 제목, 좌측 `← 목록`

- 입력 테이블: 동일한 셀 높이·폰트 크기

## 3. lesson1.html — 추세선 직접 그리기

### 3패널 레이아웃

**좌측 패널 (고정 폭)**
- x, y 데이터 입력 테이블
  - 범위: 0~100, 최대 15행
  - x, y 열 직접 입력
- [샘플 생성] 버튼: 랜덤 선형 데이터 10개 자동 생성
- [초기화] 버튼: 전체 데이터 삭제

**중간 패널 (flex 확장)**
- 좌표평면 캔버스
  - 데이터 범위에 따라 자동 줌 (auto-scale viewport)
  - 1사분면 위주 (x, y ≥ 0)
  - 격자선, 축 레이블
- 모드 전환 버튼: [점 추가 모드] / [직선 그리기 모드]
  - 점 추가 모드: 그래프 클릭 → 데이터 포인트 추가 (테이블에도 반영)
  - 직선 그리기 모드: 클릭 2점으로 직선 생성, 직선 이동·기울기 조정 가능
- 직선 여러 개 동시 표시 가능 (각 직선 방정식 레이블 표시)
- 우측 상단: ▶ 버튼 → 오른쪽 패널 열기/닫기

**우측 패널 (처음엔 숨김, ▶로 펼침)**
- 직선 선택 드롭다운 (직선이 여러 개일 때)
- 예측값·오차 표
  | x | 예측값(ŷ) | 오차(y−ŷ) |
  |---|-----------|-----------|
  | x₁ | — | — |
  | x₂ | — | — |
  - x값: 왼쪽 테이블에 입력된 x값들
  - 예측값 = 선택된 직선에서 해당 x의 y좌표
  - 오차 = 실제 y − 예측값
- [예측값 보기] 버튼: 예측값 열 표시/숨김
- [오차 보기] 버튼: 오차 열 표시/숨김

## 4. lesson2.html — y=ax 직선 비교와 MSE

### 3패널 레이아웃

**좌측 패널**
- x, y 데이터 입력 테이블
  - 범위: 0~20, 최대 15행
- [샘플 생성] [초기화] 버튼

**중간 패널**
- 좌표평면 (0~20 고정 범위)
- y=a₁x 직선 (색상 1) + y=a₂x 직선 (색상 2) 동시 표시
- 각 직선 방정식 레이블 (y = 1.5x 등)
- 잔차선(오차) 시각적 표시 (선택적)

**우측 패널**
- a₁ 슬라이더 (0~20, 소수점 1자리) + 수치 표시
- a₂ 슬라이더 (0~20, 소수점 1자리) + 수치 표시
- 손실함수 원래식 표시 (슬라이더 변경 시 실시간 갱신)
  - 처음 3항만 실제 숫자 대입, 나머지 생략
  - a₁ 기준: `(y₁−a₁x₁)² + (y₂−a₁x₂)² + (y₃−a₁x₃)² + ··· = [MSE₁값]`
  - a₂ 기준: `(y₁−a₂x₁)² + (y₂−a₂x₂)² + (y₃−a₂x₃)² + ··· = [MSE₂값]`
  - 각 합계 아래에 `÷ n = MSE` 표시
- MSE 비교: 두 값을 나란히 수치 표시 (어느 쪽이 더 작은지 강조)
- [최적 추세선 표시] 버튼 → 최소 MSE인 y=ax 직선 등장
- [최적 추세선 표시] 클릭 후 표시되는 추가 정보 박스:
  - 정리 안 된 형태의 손실함수 식:
    `MSE(a) = [(y₁−ax₁)² + (y₂−ax₂)² + ··· + (yₙ−axₙ)²] / n`
  - 최적 기울기: `a* = [값]` (소수점 2자리)
  - 최솟값: `MSE(a*) = [값]` (소수점 2자리)
  - (3차시로 연결되는 복선 역할)

## 5. lesson3.html — 손실함수와 최솟값

*(gradient.html의 수학 엔진 활용, UI 새로 구성)*

### 데이터 특성
- 샘플 데이터: y ≈ a·x 관계 (원점 근처를 지나는 직선에 노이즈 추가)로 생성
  - x, y 값 모두 양수 (0~20 범위)
  - 자연스럽게 원점 부근을 통과하는 데이터 → y=ax 모델이 적합
- 보정 설명 없음: "y=ax로 예측하는 최적 직선을 구합니다"로 안내

### 3패널 레이아웃

**좌측 패널**
- 데이터 입력 테이블 (x, y — 양수, 0~20 범위)
- [샘플 생성] 버튼: y ≈ a·x 형태의 데이터 자동 생성 (원점 근처 통과)
- [초기화] 버튼
- 산점도 캔버스 (1사분면 위주, 0~20 범위)

**중간 패널**
- a값에 따른 MSE 포물선 그래프
  - x축: 기울기 a
  - y축: MSE(a) = (1/n)Σ(yᵢ−axᵢ)²
  - 포물선 클릭 또는 슬라이더로 a값 선택 → 선택된 점 강조
  - 최솟값 위치 ★ 표시 + a* 레이블 (x축에 수직 점선)
- 좌측 산점도 캔버스와 연동: a 변경 시 추세선도 함께 갱신

**우측 패널**
- 현재 a값 (슬라이더 또는 클릭으로 변경)
- 현재 MSE(a) 값
- 손실함수 단계별 전개 (버튼 클릭 시 순서대로 등장):
  - 1단계: `MSE(a) = (1/n)[(y₁−ax₁)² + (y₂−ax₂)² + ··· + (yₙ−axₙ)²]`
  - 2단계: `= (1/n)[(Σxᵢ²)a² − 2(Σxᵢyᵢ)a + Σyᵢ²]`  ← 전개
  - 3단계: `= [A]a² − [B]a + [C]`  ← A, B, C에 실제 계산값 대입
  - [다음 단계 →] 버튼으로 순서대로 표시, 이미 나온 단계는 그대로 유지
- [최솟값 찾기] 버튼:
  - a*로 이동하는 애니메이션 (포물선 위 점이 최솟값으로 이동)
  - 이차함수 최솟값 공식 표시:
    ```
    꼭짓점: a* = B/(2A) = [값]
    최솟값: MSE(a*) = [값]
    ```
  - 활동지 연계 포인트: 학생이 직접 a* = B/(2A) 계산 후 앱으로 확인
